Lesson 2: Comparison Operators

Learn how to compare values and make decisions in your code

Introduction

Comparison operators are like judges in your code - they look at two values and decide if a statement is true or false. These operators are the foundation of decision-making in programming!

Imagine you're a bouncer at a club checking IDs. You need to compare each person's age with 21. That's exactly what comparison operators do - they help your program make decisions based on comparisons.

Basic Comparison Operators

JavaScript has six basic comparison operators that always return a boolean value (true or false):

The Comparison Toolkit:
  • > Greater than
  • < Less than
  • >= Greater than or equal to
  • <= Less than or equal to
  • == Equal to (loose equality)
  • != Not equal to

Equality Operators: == vs ===

                
JavaScript has two ways to check if values are equal, and understanding the difference is crucial!


                

                    
Loose Equality (==) vs Strict Equality (===)

                    
    
                        
  • == checks if values are equal after type conversion
    • 
                        
  • === checks if values AND types are exactly the same
    • 
                        
  • != is the opposite of ==
    • 
                        
  • !== is the opposite of ===
    • 
                    

                


                

                    

                          Best Practice:
                        
Always use === and !== unless you specifically need type conversion. This prevents unexpected bugs!

Common Mistakes to Avoid

                

                    

                        

                              Using = Instead of == or ===
                            
Remember: = assigns values, == and === compare them!

                        

                    

                    

                        

                              Comparing with NaN
                            
NaN is not equal to anything, including itself! Use isNaN() instead.

                        

                    

                    

                        

                              String Number Comparisons
                            
Remember: '10' < '2' is true because strings compare character by character!

                        

                    

                

            


            

                
Summary

                

                    
Key Takeaways:

                    
    
                        
  • Comparison operators always return true or false
    • 
                        
  • Use === and !== for strict equality without type conversion
    • 
                        
  • Use == and != only when you need type conversion
    • 
                        
  • Be careful when comparing different data types
    • 
                        
  • NaN is not equal to anything, including itself
    • 
                        
  • You can chain comparisons with logical operators (we'll learn these next!)
