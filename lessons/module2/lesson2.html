Lesson 2: let, const, and var

Three ways to declare variables, but which one should you use?

The Three Keywords

JavaScript gives us three keywords to declare variables: let, const, and var. Each has its own rules and best use cases.

let
  • Can be reassigned ✓
  • Block scoped
  • Modern (ES6+)
  • Most common choice
let age = 25; age = 26; // OK!
const
  • Cannot be reassigned ✗
  • Block scoped
  • Modern (ES6+)
  • For values that don't change
const PI = 3.14; PI = 3.15; // Error!
var
  • Can be reassigned ✓
  • Function scoped
  • Old way (avoid)
  • Has quirks
var name = "old"; name = "way"; // OK

Understanding let

let is the go-to choice for variables that might change. It's perfect for counters, user input, or any value that updates.

Understanding const

const creates constants - variables that cannot be reassigned. Use it for values that should never change throughout your program.

Important: const prevents reassignment, not modification. You can still change the contents of objects and arrays declared with const.

When to Use Each

Use const by default

Start with const for all variables. This prevents accidental reassignment.

const userName = "Alice"; const MAX_ATTEMPTS = 3; const config = { theme: "dark" };

Use let when you need to reassign

If the value needs to change, switch from const to let.

let counter = 0; let currentUser = null; let isLoading = true;

Never use var

There's no good reason to use var in modern JavaScript. Always use let or const.

var oldWay = "avoid";

Best Practices Summary

✅ DO

  • Use const by default
  • Use let only when reassignment is needed
  • Declare variables at the top of their scope
  • Use descriptive variable names

❌ DON'T

  • Never use var
  • Don't reassign const variables
  • Don't declare without let or const
  • Avoid global variables when possible
