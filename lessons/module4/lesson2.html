Lesson 2: Else and Else If

Handle multiple paths and create complete decision trees

Introduction

In the last lesson, you learned how to make your code do something IF a condition is true. But what if you want to do something different when the condition is false? Or what if you have multiple conditions to check? That's where else and else if come in!

Think of it like choosing what to wear based on the weather. If it's raining, you wear a raincoat. Else if it's cold, you wear a jacket. Else (if it's neither), you wear a t-shirt. Your code can make these same kinds of multi-path decisions!

The Else Statement

The else statement provides an alternative path when the if condition is false. It's like saying "if this, do that, otherwise do this other thing."

The If-Else Pattern: 
if (condition) {
    // Code runs if condition is true
} else {
    // Code runs if condition is false
}

The Else If Statement

                
Sometimes you need to check multiple conditions in sequence. The else if statement lets you test additional conditions when the first if is false.


                

                    
The Complete If-Else If-Else Chain:

                    
if (condition1) {
    // Runs if condition1 is true
} else if (condition2) {
    // Runs if condition1 is false AND condition2 is true
} else if (condition3) {
    // Runs if condition1 and condition2 are false AND condition3 is true
} else {
    // Runs if all conditions are false
}

Order Matters!

                
When using else if statements, the order of conditions is crucial. JavaScript checks conditions from top to bottom and stops at the first true condition.


                

                    

                          Watch the Order!
                        
Always put more specific conditions first. If a general condition comes first, the specific ones will never be reached!

Common Mistakes to Avoid

                

                    

                        

                              Wrong Condition Order
                            
Put specific conditions before general ones, or they'll never be reached!

                        

                    

                    

                        

                              Missing Else
                            
Always consider what happens when all conditions are false. Add an else for completeness.

                        

                    

                    

                        

                              Too Many Else Ifs
                            
If you have many else ifs (5+), consider using a switch statement instead.
