Lesson 1: If Statements

Learn to make your code think and decide

Introduction

Imagine you're a robot following instructions. So far, you've been following every instruction in order, no matter what. But what if you need to make a decision? What if you should only do something when it's raining, or when a user is logged in? That's where if statements come in!

An if statement is like a bouncer at a club - it checks if a condition is met before allowing the code inside to run. It's the fundamental building block of decision-making in programming.

Basic If Statement

The if statement has a simple structure: it checks a condition, and if that condition is true, it runs the code inside the curly braces.

The If Statement Recipe: 
if (condition) {
    // Code to run if condition is true
}

If Statements with Expressions

                
The condition in an if statement can be any expression that evaluates to true or false. You can use variables, comparisons, and even function calls!


                

                    
What Makes a Condition?

                    
    
                        
  • Comparison expressions: x > 5, name === "John"
    • 
                        
  • Boolean variables: isLoggedIn, hasPermission
    • 
                        
  • Function calls that return boolean: isValidEmail(email)
    • 
                        
  • Logical expressions: age >= 18 && hasID

Code Blocks and Scope

                
The curly braces {} create a code block. Variables declared inside this block are only accessible within it - this is called "block scope".


                

                    

                          Important: Always Use Braces!
                        
While JavaScript allows single-line if statements without braces, always use them. It prevents bugs and makes code clearer.

Nested If Statements

                
You can put if statements inside other if statements. This is called "nesting" and allows you to check conditions that depend on other conditions.


                

                    
 Pro Tip: Avoid Deep Nesting

                    
While nesting is powerful, too many levels make code hard to read. Try to keep nesting to 2-3 levels maximum.

Common Mistakes to Avoid

                

                    

                        

                              Using = Instead of ===
                            
Remember: = assigns values, === compares them! if (x = 5) will always be true!

                        

                    

                    

                        

                              Forgetting Curly Braces
                            
Always use braces, even for single lines. It prevents bugs when adding more code later.

                        

                    

                    

                        

                              Over-Nesting
                            
Too many nested ifs make code hard to read. Consider restructuring or using early returns.

                        

                    

                

            


            

                
Summary

                

                    
Key Takeaways:

                    
    
                        
  • If statements let your code make decisions based on conditions
    • 
                        
  • The condition must evaluate to true or false
    • 
                        
  • Always use curly braces {} to define the code block
    • 
                        
  • You can use any expression that returns a boolean as a condition
    • 
                        
  • If statements can be nested inside each other
    • 
                        
  • Variables declared inside an if block have block scope
